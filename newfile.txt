

1. git status -> gives the status of the files in the current repo.
2. git add . -> adds files for tracking and staging, where tracking refers to the act of making git watch the changes made in a file and staging refers to the act of making the changes made in a tracked file eligible for commit. Note that only tracking and staged changes are committed.
3. git status
4. git commit -m "" -> creates a snapshot of the current state of the repository.
5. git log -> shows a list of the commits made in the repo.

Branching:
1. git branch -> shows a list of branches in a repository and also marks the current branch.
2. git checkout <branchname> -> switches to <branchname>
3. git checkout -b <newbranchname> -> creates a new branch called <newbranchname> and switches to it.
